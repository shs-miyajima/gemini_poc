### 5.3 セキュリティ観点チェック

- [ ] 認可の担保箇所を明記した（どの層でロール・所有者判定をするか。フロントの表示制御だけに頼らない）
- [ ] mass assignment 対策（Model の `fillable` に必要最小限のみ列挙）
- [ ] XSS エスケープの担保箇所を明記した（Blade の `{{ }}` / JS で描画する場合はエスケープ関数）
- [ ] SQL インジェクション（生クエリ・`whereRaw` を使う場合はバインド必須。未使用なら「なし」）
- [ ] CSRF（フォーム・axios の CSRF トークン設定）
