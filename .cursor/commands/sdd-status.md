# SDD 進捗一覧

`docs/specs/` 配下の全機能の SDD 進捗を一覧表示する。**読み取り専用**（ファイルの変更・作成はしない）。

## 手順

1. `docs/specs/` 直下のディレクトリを列挙する（`_templates/` は除外）
2. 各 slug について以下を読み取る:
   - `meta.yaml` の `display_name`
   - `01-requirements.status` / `02-design.status` / `03-test-plan.status`（1 行目の状態と `date:`）
3. 以下の表形式で表示する:

| slug | 機能名 | 1.仕様整理 | 2.設計 | 3.テスト設計 | 現在のフェーズ |
|------|--------|-----------|--------|-------------|----------------|

4. 「現在のフェーズ」は最初に `approved` でないフェーズ（すべて approved なら「4.実装」）
5. `rejected` のフェーズがある場合は `changelog.md` の直近の差戻し理由も添える
6. `effort-report.md` がある機能は、人手想定工数・実績時間・削減率（§3 が記入済みの場合）も表に添える
